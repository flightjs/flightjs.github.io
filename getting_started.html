Getting started with Flight

This tutorial will cover all of the necessary steps to start building web applications with Flight.

Prerequisites

  • Make sure Node.JS and npm are installed on your machine [download installers here]
  • Make sure Bower is installed globally via npm [instructions here]

Installation

Installation can be done with Bower, a JS package manager. There is also a standalone version available, but there are a few caveats:

  • The standalone version does not have support for module loading systems, such as requireJS. It instead instantiates global variables on the window object.
  • The standalone version does not manage dependencies for you; you will need to download and configure access to your Flight's dependencies yourself.

Overall, it is recommended that you install Flight either by using Bower or by using Yeoman to scaffold your project for you. Both of these installation methods are detailed in this tutorial.

Installation via Bower

Installing Bower

If you haven't yet, install Node.JS, which will install npm.

Then run the following command:

        
        $ npm install -g bower

Installing Flight

cd into your project directory. Then run the following:

        
        $ bower install flight

This will not only install Flight, but all of Flight's dependencies, into a bower_components folder in your project's directory.

Installation via Yeoman and flight-generator

Installing Yeoman

If Yeoman isn't already installed, run the following:

        
        $ npm install -g yo

Installing generator-flight

The way Yeoman works is it uses a generator, which contains a template for a project. These generators are installed via npm. The generator we'll be using is provided by the flight maintainers, and it is called generator-flight. In order to install generator-flight, run the following:

        
        $ npm install -g generator-flight

Running the Flight Yeoman Generator

NOTE: Be aware that if your project folder is not empty when you run Yeoman with the Flight generator, it could cause issues or errors with your project's scaffolding. This method should be used with an empty project directory.

In your project directory, run the following:

        
        $ yo flight

Yeoman will ask you a series of questions, including whether to include Bootstrap, Bootstrap Theme, and a Google Analytics snippet. Answer these questions and Yeoman will get to work scaffolding an app for you.

Writing a Component

A Flight component consists of a few simple parts:

  • Default Attributes
  • Initialization
  • Event Handlers
  • Component Methods

Basic Scaffolding for a Component

This is the basic skeleton of a component before the parts are added:

define(require){
            //pull in the Flight module that encapsulates your code into a component
            var createComponent('bower_components/flight/lib/component');

            function myComponent(){
                //this will be reference as the 'component function' from here on out.
            }

            return createComponent(myComponent);
        }

This creates an AMD module containing a Flight component.

Default Attributes

Calling this.DefaultAttrs({}); within a component function, you can set default attributes for your component. Flight offers some functionality with these attributes, especially if they are jquery selectors.

What should go in defaultAttrs()? At the very least, your node's child selectors. For example, if you're wrapping the following html in a Flight component:

<form class="myForm">
            <div class="success-message"></div>
            <label for="my-name">My Name:</label>
            <input type="text" name="my-name" placeholder="Jane Doe" />
            <label form="my-email">My Email:</label>
            <input type="text" name="my-email" placeholder="jdoe@gmail.com" />
            <input type="submit" value="Submit" />
        </form>

Your defaultAttrs() will look like this:

this.defaultAttrs({
            "successMessage": ".success-message",
            "myName": "input[name=my-name]",
            "myEmail": "input[name=my-email]" 
        });

This allows us to utilize several functions that are given to us by the framework.

If you wish to override these defaults for a specific instance, you can do so when you attach the component (covered in Using a Component).

Initialization

The initialization of a component is where you handle any setup for your component instance, including your event listeners.

To set up your initialization, add this to your component function:

this.after('initialization', function(){
            //the this variable references the component instance
        });

In this initialization handler, we'll set up our event listeners for our component, using the given method:

this.on(selector, 'event', handler);

Your handler should be this. something. This gives you access to the component instance via this this variable.

Event Handlers/Component Methods

These handle the bulk of the work for your component. They are built on top of the this variable in your component function

this.onEvent = function(event, data){
            //your component logic

            this.trigger(selector, eventType, data);
        };

The difference between component functions and event handlers is that the component functions aren't called by listeners. But they still have access to functions available to components if they are attached the the this variable.

Using a Component

Once you have set up your component, you get it running by attaching it to an html fixture on the page. In your main JS file, you'll pull it in using require, and then call .attachTo() on it.

require([
            'myComponent'
        ],
        function(
            myComponent
        ){
            myComponent.attachTo(selector);
        });

If you wish to override your default attributes, this would be the time to do it. You can pass a second argument object to do so:

myComponent.attachTo({
            "successMessage": ".my-other-success-message"
        });
